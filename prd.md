# PDF编辑器 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
开发一款 Windows 桌面程序，支持把多个 PDF 合并成一个、删除单个PDF多余页面、调整页面顺序、维护文档目录，或者把 PDF 内容导出成 Word、PPT、Excel 等常用格式。

### 1.2 目标用户
主要面向日常需要处理 PDF 文件的办公用户，例如行政、人事、财务、教师、学生、项目管理人员，以及经常整理报告和资料的职场用户。

### 1.3 技术架构
- **开发框架**：.NET 8.0 + WPF
- **PDF 处理**：iTextSharp 5.5.13.3
- **Word 导出**：Xceed.Words.NET (DocX)
- **Excel 导出**：EPPlus 5.8.10
- **PPT 导出**：DocumentFormat.OpenXml 3.0.2
- **PDF 预览**：WebView2 + PDF.js

## 2. 功能需求

### 2.1 已完成功能

#### 2.1.1 PDF 合并
- 支持选择多个 PDF 文件
- 支持调整合并顺序（上移/下移）
- 支持移除不需要的文件
- 进度条显示合并进度
- 合并完成后自动保存

#### 2.1.2 页面编辑
- 打开单个 PDF 文件
- 显示页面列表
- **页面预览**：左侧选择页面，右侧实时预览对应内容
- 删除不需要的页面
- 调整页面顺序（上移/下移）
- 保存编辑后的 PDF

#### 2.1.3 格式导出
- **Word 导出**：将 PDF 文本内容导出为 .docx 格式
- **Excel 导出**：将 PDF 文本内容按行导出为 .xlsx 格式
- **PPT 导出**：将 PDF 文本内容导出为 .pptx 格式

#### 2.1.4 目录维护
- 添加书签（标题 + 页码）
- 编辑已有书签
- 删除书签
- 支持多级书签
- 将书签保存到 PDF 文件

## 3. UI/UX 设计

### 3.1 整体布局
采用多标签页设计，分为四个主要功能区：

1. **PDF 合并** - 左侧文件列表 + 右侧操作按钮
2. **页面编辑** - 左侧页面列表 + 右侧预览区域
3. **格式导出** - 导出选项 + 导出按钮
4. **目录管理** - 书签树形视图 + 编辑按钮

### 3.2 组件设计
- 使用原生 Windows 控件，确保用户体验一致
- 状态栏显示当前操作状态
- 统一的文件对话框和消息提示
